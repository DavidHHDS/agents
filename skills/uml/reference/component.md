# 3. Diagrama de Componentes (Teoría)

### Teoría
Muestra la organización y las dependencias entre los componentes de software a nivel macro o arquitectónico (módulos, librerías, bases de datos, APIs).

### Relaciones
* **Interfaz Proporcionada (`()-`):** El componente ofrece un servicio hacia afuera (API pública).
* **Interfaz Requerida (`)-(`):** El componente necesita un servicio de otro componente para funcionar.
* **Dependencia (`..>`):** Un componente depende lógicamente de otro para conectar interfaces.
